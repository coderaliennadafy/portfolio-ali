/ project overview

Manual Testing — nopCommerce

I performed extensive manual testing for the nopCommerce platform, focusing on the user journey from product discovery to successful checkout.

/ test coverage

Module-by-Module Testing

01. Registration

Tested the full user onboarding flow. Validated mandatory fields (Email, Password), account activation links, and error handling for duplicate emails or weak passwords to ensure only valid users can join the platform.

02. Login

Verified secure access using multi-role credentials. Tested "Remember Me" functionality, password recovery via email, and account locking mechanisms after multiple failed attempts to protect user data.

03. Logout & Session

Ensured that user sessions are properly terminated upon logout. Verified that back-button navigation doesn't expose sensitive account pages once the user has signed out.

04. Session Expiry

Validated automatic timeout for inactive sessions. Tested if the system redirects users to the login page after a period of inactivity to maintain e-commerce security standards.

05. Product Listing

Tested the display accuracy of products across categories. Verified grid/list view toggles, pagination, and ensured that out-of-stock items are labeled correctly to avoid customer confusion.

06. Product Details

Verified that all specifications, prices, and high-res images load correctly. Tested variant selections (Size/Color) and ensured the "Add to Wishlist" and "Compare" features function as expected.

07. Search & Filter

Tested the search algorithm for keyword accuracy. Validated advanced filters (Price Range, Brand, Rating) to ensure customers can find specific products quickly in a large catalog.

08. Shopping Cart

Performed functional testing on quantity updates, item removal, and subtotal calculations. Verified that discount codes are applied correctly and reflect immediately in the total price.

09. Checkout

The most critical flow. Tested the 6-step checkout process, including billing/shipping address validation, shipping method selection, and secure payment gateway integration.

10. Order Confirmation

Verified the generation of Order IDs and success messages. Checked if the system sends a confirmation email with the correct order summary and receipt details to the user.

11. Order History

Tested the "My Account" order section. Verified that users can track their order status (Pending/Processing/Shipped) and re-order previous items with a single click.

12. User Profile

Tested the ability to update personal information, change passwords, and manage multiple shipping addresses. Verified data persistence across different devices.

/ takeaways

What I Learned

The QA Mindset

As my first project in manual testing, I learned that testing isn't just about finding bugs — it's about understanding the user's journey. I realized that a successful tester needs to think like a customer while maintaining the technical rigor of a developer.

Test Case Prioritization

Managing 280+ test cases taught me the importance of prioritization. I learned how to identify "Smoke" and "Sanity" tests to ensure that the most critical features (like Checkout and Payment) are stable before diving into minor UI details.

Attention to Detail

I discovered that even the smallest misalignment in a UI or a missing validation message in a form can impact user trust. This project sharpened my eye for detail, especially when testing responsive layouts on different screen sizes.

Reporting & Documentation

I learned that a bug is only as good as its report. Documenting clear, reproducible steps and expected results in Google Sheets taught me how to communicate effectively with stakeholders and keep a project organized.
